#vr_pt:
API-led é uma abordagem de integração que organiza as APIs em três camadas: Experience, Process e System.  
Essa estrutura facilita a reutilização de componentes, acelera o desenvolvimento e reduz a duplicação de esforço.  
Ao dividir as responsabilidades entre as camadas, as equipes conseguem manter as integrações de forma mais eficiente.  
Além disso, o modelo permite maior controle, governança e segurança sobre os serviços expostos.  
É um dos principais fundamentos da arquitetura MuleSoft para conectar sistemas de forma escalável e organizada.
#vr_en:
API-led is an integration approach that organizes APIs into three layers: Experience, Process, and System.  
This layered model promotes reusability, speeds up development, and reduces redundancy.  
By separating concerns across layers, teams can manage integrations more effectively.  
It also allows for better governance, security, and control over exposed services.  
API-led is a foundational principle in MuleSoft architecture for scalable and organized system connectivity.

A camada **Experience** é a camada mais próxima do consumidor final em uma arquitetura API-led.  
Ela é responsável por **apresentar os dados** de forma personalizada para diferentes canais, como web, mobile ou parceiros.  
Sua principal função é **adaptar e expor** as informações de outras camadas (Process e System) de forma acessível.

É a interface pública da API, focada na experiência do usuário e na entrega personalizada de valor.
#vr_en:
The **Experience Layer** is the closest layer to the end-user in an API-led architecture.  
It is responsible for **presenting data** in a personalized way across different channels like web, mobile, or partners.  
Its main role is to **adapt and expose** information from other layers (Process and System) in an accessible format.  
This layer enables reuse of core logic without duplicating code in every consumer channel.  
It acts as the public interface of the API, focused on user experience and tailored value delivery.

#vr_pt:
A camada **Process** é responsável por orquestrar os fluxos de dados entre sistemas e realizar o processamento necessário.  
Ela lida com a **lógica de negócios**, validando, transformando e direcionando dados entre as camadas Experience e System.  
Essa camada é fundamental para garantir que os dados certos sejam entregues da maneira correta para o usuário ou sistema final.  
Além disso, a camada Process também é responsável pela **integração** com outros sistemas e pelo **gerenciamento de erros** e falhas.  
É a camada que atua diretamente na **lógica de transformação e integração**, conectando os sistemas e controlando o fluxo de dados.
#vr_en:
The **Process Layer** is responsible for orchestrating data flows between systems and performing the necessary processing.  
It handles **business logic**, validating, transforming, and routing data between the Experience and System layers.  
This layer is key to ensuring that the correct data is delivered in the proper format to the end user or system.  
In addition, the Process Layer is also responsible for **integration** with other systems and **error handling**.  
It acts directly on the **logic of transformation and integration**, connecting systems and managing data flow.

#vr_pt:
A camada **System** é responsável por **gerenciar os sistemas internos**, como bancos de dados, aplicações legadas e outros serviços de back-end.  
Ela lida com a **integração e persistência** de dados, conectando as fontes de dados com a camada Process para que as informações sejam acessíveis e utilizáveis.  
Essa camada também é responsável por **gerenciar a comunicação entre sistemas** e garantir que a infraestrutura de TI suporte as integrações.  
Além disso, a camada System oferece **segurança e consistência** ao garantir que as APIs estejam conectadas corretamente e que as trocas de dados sejam realizadas de forma eficiente.  
É a camada mais voltada à **infraestrutura e ao processamento de dados** dentro da arquitetura MuleSoft.
#vr_en:
The **System Layer** is responsible for **managing internal systems**, such as databases, legacy applications, and other back-end services.  
It handles **data integration and persistence**, connecting data sources to the Process layer so that information is accessible and usable.  
This layer is also responsible for **managing communication between systems** and ensuring that the IT infrastructure supports the integrations.  
Additionally, the System Layer provides **security and consistency** by ensuring that APIs are correctly connected and that data exchanges are efficient.  
It is the layer most focused on **infrastructure and data processing** within the MuleSoft architecture.
